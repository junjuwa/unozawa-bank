# おこづかい管理アプリ — Claude Code ハンドオフ

## 0. これは何 / 渡し方
子供向けおこづかい管理アプリ（兄弟2人 れい・じゅん＋親）の **高忠実度デザイン** とその実装仕様です。
- **デザインリファレンス**：`Okozukai-Home.html`（単一HTML・全画面が1キャンバスに並ぶ）。ブラウザで開けばそのまま見られます。
- Claude Code への渡し方：このリポジトリ（`Okozukai-Home.html` ＋ 本 `HANDOFF.md` ＋ `image-slot.js`）を Claude Code に渡し、「`HANDOFF.md` を読み、`Okozukai-Home.html` をビジュアル基準として React/React Native で実装して」と指示してください。**ピクセルは HTML を正とし、本書は構造・トークン・ルール・データモデルの正とします。**

## 1. プロダクト要点
- お金は3つの箱で管理：**つかう / ためる / ふやす**。
- **ためる**＝目標（ほしいもの）に向けて貯金。複数目標を持てる。
- **ふやす**＝30日の投資。満期で利息がつき **ためる** へ自動移動。
- 子供＝6〜7歳。漢字はほぼ読めない前提。**漢字＋ルビ（3年生までの配当漢字のみ）**。4年以上は使わずやさしい語に言い換え。

## 2. 一貫性ルール（最重要）
- **骨格（レイアウト・ナビ・カード配置・操作フロー）は全画面・全ユーザー共通。** 変えない。
- ユーザーで差し替わるのは **表層トークンのみ**（配色・質感・角丸・装飾・タイポ）。同一ユーザーは全画面で同一テイスト。
  - **れい**＝水彩トロピカル（全画面）
  - **じゅん**＝アメコミヒーロー（全画面）
  - **親**＝ダークグレー管理UI
- 状態は **色だけで表さない**。必ずアイコン／形を併用。コントラスト確保。

## 3. 画面一覧（HTML内 `data-screen-label` で対応）
子供（れい・じゅんで同一骨格、表層のみ差し替え）：
1. **ホーム**：ヘッダー（アバター＋名前＋所持金）→ マスコット帯 → ためる大カード（目標プログレス＋「ほかに◯こ」）＋つかう/ふやす → ボトムナビ5本。
2. **ためる（もくひょう いちらん）**：目標カード一覧（絵＋名前＋進捗＋あと◯えん＋「いま ためてる／つぎ」＋追加）。ホームの「ためる」タップで遷移（ナビ項目ではない）。
3. **ふやす**：ロットごと「あと◯かい ねたら」＋月の列＋プログレス＋満期額。
4. **ふりかえ（振替）**：から箱→へ箱の選択＋数ステッパー＋確認ボタン。
5. **おしごと**：絵カード＋報酬＋状態（しんせいちゅう／申請する／OK！承認）。
6. **やくそく（ルール）**：アイコン＋漢字＋ルビのカード。
- 端末展開：スマホ390×844、iPad縦834×1112（ボトムナビ維持・カードは自動多列）、iPad横1194×834（左サイドナビ＋ワイドコンテンツ）。**コンテンツは `repeat(auto-fill, minmax(...))` で端末幅に追従**（端末ごとの分岐を最小化）。

親（iPhone 390×844・ダークグレー・漢字表記・ボトムナビ3本：ダッシュボード/承認/設定）：
7. **ダッシュボード**：KPI（合計残高・承認待ち・今週貯めた）＋兄弟別カード（残高・3箱内訳・目標達成度・今週ミニグラフ）。
8. **承認**：兄弟フィルタ＋申請カード（却下／承認する）＋**承認履歴（日時付き）**。
9. **設定**：基本給（兄弟別・頻度）／ふやす（利率・満期日数＋計算例）／お仕事の単価。

## 5. 文字表記・ルビ方針
- `<ruby>漢字<rt>よみ</rt></ruby>`。**rt は本文の約50%サイズ**。行の高さは最初からルビ込みで設計（後付けで詰めない）。
- ルビは **「文を読む」要素**（目標名・ルール文・あいさつ・「円」など）に限定。アイコン＋短語で意味が取れるナビ・箱名（つかう/ためる/ふやす）には振らない。
- 同一画面で繰り返す同語は **初出のみルビ**。
- **将来「ぜんぶひらがな」モードを追加できる前提**：文言はハードコードせず、表記レベルを切替できるデータ（例 `t(key, level)`）として持つ。親・設定に表記トグルのUIあり。

## 6. 権限・ルール（UIに反映済み）
- 子供のアクションは **自分の箱の間の振替だけ**。子供画面に「支出」「購入」ボタンは置かない。
- 目標達成時、子供画面は **お祝い演出のみ**（実際の購入＝引き落としは親が行う）。
- お仕事：子供が申請 → 親が承認/却下 → 承認時に単価が **ためる** へ加算。
- ふやす：満期（既定30日）で利率（既定5%）の利息がつき **ためる** へ移動。

## 7. データモデル（実装の目安）
```
User { id, role: 'child'|'parent', displayName, theme: 'rei'|'jun', avatarUrl }
Box  { userId, kind: 'use'|'save'|'invest', balance }            // 3箱/ユーザー
Goal { id, userId, name, imageUrl, targetAmount, savedAmount, active: bool, order }
Lot  { id, userId, principal, startDate, termDays, rate, maturesAt, status }  // ふやす
Job  { id, name, unitPrice }                                      // 単価マスタ
JobRequest { id, childId, jobId, amount, status:'pending'|'approved'|'rejected', requestedAt, decidedAt }
Transfer { id, userId, fromKind, toKind, amount, createdAt }
Settings { childId, basePay, payFrequency, investRate, investTermDays, textLevel:'kanji-ruby'|'hiragana' }
```
- 金額は整数（円）。利息は満期時 `round(principal * rate)` を save 箱へ。

## 8. フィードバック（演出）
入金＝お金の効果音、目標達成・満期＝紙吹雪、コイン（お金の絵）が動く確認演出。各テーマにマスコット枠（案内・応援、短文吹き出し）。**楽しいが煽らない**（不安・競争で煽らない）。

## 9. アセット / 注意
- **アバター・マスコット・目標の絵は画像枠（`<image-slot>`）**：ユーザーが自分の画像を差し込む。`id` 単位で永続化（同一IDは全画面連動）。React実装では画像URLフィールド＋アップロード/差し替えに置換。
- **実在キャラクター（スティッチ／スパイダーマン等）は使用・再現・名指し禁止。** あくまで「水彩トロピカル」「アメコミヒーロー」という画風・配色の方向性として表現する。
- アイコンは単純図形（円・三角・矩形）主体のインラインSVG。金額は数字＋お金の絵を併記し桁感覚を補う。

## 10. 実装の進め方（推奨）
1. テーマトークンを定義（`theme.rei/jun/parent`）。
2. 共通骨格コンポーネント：`AppShell`（ヘッダー＋コンテンツ＋ボトムナビ/サイドナビ）、`BalanceCard`、`GoalCard`、`LotCard`、`TransferFlow`、`JobCard`、`RuleCard`、`Coin`、`ProgressBar`、`RubyText`。
3. 子供フローを `theme` 切替で2テイスト確認 → 親フロー。
4. レスポンシブは `auto-fill/minmax` ＋ 横=サイドナビ/縦=ボトムナビ の分岐のみ。
5. i18n/表記レベルを `t(key, level)` で抽象化（ひらがなモード対応の布石）。
